gdb调试php

php进程进入死循环，php出现core文件等，是业务场景中偶尔会出现的问题。如何快速有效定位到问题所在，是我们解决问题的关键。用gdb对其进行调试，是个非常不错的手段。

调试死循环

如果php进程长期处于R状态，那么可能是进入了死循环，可通过下面示例分析：

gdb -p pid      // pid为进程id
source gdbinit  // gdbinit是php自带的一个工具
bt              // 打印堆栈信息
zbacktrace      // 代码执行过程
printzv core_globals  
printzv sapi_globals

调试core文件

线上出现core文件是一个比较严重的问题，用gdb来分析core文件是个不错的选择。调试示例如下：

gdb php -c core.222 // core.222是一个core文件
source gdbinit
bt 
p **
zbacktrace

如果手动生成core文件，可能需要设置core文件大小不受限制，linux系统下命令：ulimit -c unlimited

几种加密算法

下面给出几种加密算法，有的可逆，有的不可逆，用于不同的场景。

签名

该签名算法将参数排序，加上密钥，然后计算md5值，该算法不可逆。
该算法可用于以下情况：1）网络通信，通信双方约定密钥，对于请求参数进行加密，用于验证请求正确性；2）颁发token，密钥自己保留，对方拿token请求自己的数据，用于验证权限等。
算法的php实现如下：

function getSign($params, $secret, $sign_key = 'sign')
{
    $str = '';
    ksort($params);
    foreach ($params as $k => $v) {
        if ($k != $sign_key) {
            $str .= "$k=$v";
        }
    }
    $str .= $secret;
    return md5($str);
}

id可逆加密

该算法用于id加密，算法可逆，能反解出id。该算法目的是保护真实id，将加密后id返给使用者。
该算法的php实现如下：

// 加密算法
function encodeId($id)
{
    $sid  = ($id & 0x0000ff00) << 16;
    $sid += (($id & 0xff000000) >> 8) & 0x00ff0000;
    $sid += ($id & 0x000000ff) << 8;
    $sid += ($id & 0x00ff0000) >> 16;
    $sid ^= 818329; //该值定了就不能再改了，否则就出问题了
    return $sid;
}
// 解密算法
function decodeId($sid)
{
    if (!is_numeric($sid)) {
        return false;
    }
    $sid ^= 818329; //该值定了就不能再改了，同加密算法，否则就出问题了
    $id   = ($sid & 0x00ff0000) << 8;
    $id  += ($sid & 0x000000ff) << 16;
    $id  += (($sid & 0xff000000) >> 16) & 0x0000ff00;
    $id  += ($sid & 0x0000ff00) >> 8;
    return $id;
}

对称加密

该算法可用于数据加密，是一种对称加密，能保护数据安全。
该算法的php实现如下：

// 加密
function encrypt($id, $secret)
{
    $md5_v = md5($secret);
    // Open the cipher
    $td = mcrypt_module_open('rijndael-128', '', 'cbc', '');
    // Create key and IV
    $key = substr($md5_v, 0, 16);
    $iv  = strrev(substr($md5_v, 0, 16));
    // Intialize encryption，不满16字符后面补\0
    mcrypt_generic_init($td, $key, $iv);
    // Encrypt data
    $id = mcrypt_generic($td, trim($id));
    // Terminate encryption handler and close module
    mcrypt_generic_deinit($td);
    mcrypt_module_close($td);
    return base64_encode($id);
}
// 解密
function decrypt($id, $secret)
{   
    $id = base64_decode($id);
    if (empty($id)) {
        return false;
    }
    $md5_v = md5($secret);
    // Open the cipher
    $td = mcrypt_module_open('rijndael-128', '', 'cbc', '');
    // Create key and IV，不满16字符后面补\0
    $key = substr($md5_v, 0, 16);
    $iv = strrev(substr($md5_v, 0, 16));
    // Intialize encryption
    mcrypt_generic_init($td, $key, $iv);
    // Encrypt data
    $id = mdecrypt_generic($td, $id);
    // Terminate encryption handler and close module
    mcrypt_generic_deinit($td);
    mcrypt_module_close($td);
    return trim($id);
}

php日志监控

项目中没有对php日志进行监控，导致php fatal等处理不及时，自己写一个shell脚本，通过定时任务，对php日志进行监控，并发邮件给相应人员。

#!/bin/bash

#php日志目录
dir='php/'
log_f=$dir"php.log"
line_f=$dir"line.log"
#我们的php日志是按照天进行切割，所以需要一个记录天的文件
day_f=$dir"day.log"

today=`date +%Y%m%d`
if [ -f "$day_f" ];then
    lastday=`cat $day_f`
fi

total_l=`awk 'END{print NR}' $log_f`
if [ -f "$line_f" ];then
    last_l=`cat $line_f`
fi

if [[ "$today" -eq "$lastday" ]]; then
    read_l=$((total_l-last_l))
else    
    read_l=$total_l
fi

if [[ $read_l -gt 0 ]]; then
    echo "$total_l" >"$line_f"
    echo "$today" >"$day_f"
    content=`tail -n $read_l $log_f |grep -E "Warning|Fatal|error|PHP|#|Stack|trace|Exception|session|main|object|static|property|class|method|function|mysql|redis|mongo"`
    echo $content | mail -s 'PHP 日志监控' guanhua2011@gmail.com
fi
