gdb调试php

php进程进入死循环，php出现core文件等，是业务场景中偶尔会出现的问题。如何快速有效定位到问题所在，是我们解决问题的关键。用gdb对其进行调试，是个非常不错的手段。

调试死循环

如果php进程长期处于R状态，那么可能是进入了死循环，可通过下面示例分析：

gdb -p pid      // pid为进程id
source gdbinit  // gdbinit是php自带的一个工具
bt              // 打印堆栈信息
zbacktrace      // 代码执行过程
printzv core_globals  
printzv sapi_globals

调试core文件

线上出现core文件是一个比较严重的问题，用gdb来分析core文件是个不错的选择。调试示例如下：

gdb php -c core.222 // core.222是一个core文件
source gdbinit
bt 
p **
zbacktrace

如果手动生成core文件，可能需要设置core文件大小不受限制，linux系统下命令：ulimit -c unlimited
