redis有序集根据key排序问题

redis 有序集达不到预期排行的问题

游戏中经常用redis的有序集做排行，排行的key采用uid，排序积分用游戏数值。不同玩家，游戏数值经常相同，理想情况下，应该是早达到该数值的玩家排行在前。
由于redis有序集具有排行积分相同的，按照key进行排序的特性，我们的排序总是达不到预期的结果。
比如下面一个示例:

redis 127.0.0.1:53000> del test
(integer) 1
redis 127.0.0.1:53000> zadd test 10 1
(integer) 1
redis 127.0.0.1:53000> zadd test 10 3
(integer) 1
redis 127.0.0.1:53000> zadd test 10 2
(integer) 1
redis 127.0.0.1:53000> zrange test 0 -1 withscores
1) "1"
2) "10"
3) "2"
4) "10"
5) "3"
6) "10"

其实我们期望的排序应该是1、3、2。

排行积分中增加时间解决该问题

我们期望越早进入排行的玩家排行在前，如果用redis的有序集做排行，仅用游戏数值做排行积分无法满足我们的需求，考虑到时间因素，因而想到将时间加入的排行积分也就水到渠成。
具体的算法以下面的周排行为例：

function getRedisPoint($point)
{
    $_t    = time();
    $index = date('N', $_t);
    $begin = strtotime(date('Y-m-d', $_t));
    $min   = ($index - 1) * 1440 + ceil((SVR_NOW - $begin) / 60);
    return $point * 20000 - $min;
}

上例为计算redis周排行的一个示例，1440是一天的分钟数（60*24），一周总共10080，为计算方便采用20000，当然可以选择大于10080的任意数值。
计算真正积分的算法如下：

function getRealPoint($point)
{
    return $point > 20000 ? ceil($point / 20000) : $point;
}

其他的排行也类似，不过是选取不同的常数。

iTerm/Term/vim配色方案

小伙伴们应该也对编辑器的配色头疼，并不时地更新自己的配色。个人也比较头疼，最终找到一个不错的配色方案，支持iterm、vim、emacs、netbeans等等。
给出github地址：https://github.com/altercation/solarized。

Terminal/iTerm2

Mac OS X 自带的 Terminal 和免费的 iTerm2 都是很好用的工具，iTerm2 可以切分成多窗口，更方便一些。
如果你使用的是 Terminal 的话，在 solarized/osx-terminal.app-colors-solarized 下双击 Solarized Dark ansi.terminal 和 Solarized Light ansi.terminal 就会自动导入两种配色方案 Dark 和 Light 到 Terminal.app 里。
如果你使用的是 iTerm2 的话，到 solarized/iterm2-colors-solarized 下双击 Solarized Dark.itermcolors 和 Solarized Light.itermcolors 两个文件就可以把配置文件导入到 iTerm 里。

php日志监控

项目中没有对php日志进行监控，导致php fatal等处理不及时，自己写一个shell脚本，通过定时任务，对php日志进行监控，并发邮件给相应人员。

#!/bin/bash

#php日志目录
dir='php/'
log_f=$dir"php.log"
line_f=$dir"line.log"
#我们的php日志是按照天进行切割，所以需要一个记录天的文件
day_f=$dir"day.log"

today=`date +%Y%m%d`
if [ -f "$day_f" ];then
    lastday=`cat $day_f`
fi

total_l=`awk 'END{print NR}' $log_f`
if [ -f "$line_f" ];then
    last_l=`cat $line_f`
fi

if [[ "$today" -eq "$lastday" ]]; then
    read_l=$((total_l-last_l))
else    
    read_l=$total_l
fi

if [[ $read_l -gt 0 ]]; then
    echo "$total_l" >"$line_f"
    echo "$today" >"$day_f"
    content=`tail -n $read_l $log_f |grep -E "Warning|Fatal|error|PHP|#|Stack|trace|Exception|session|main|object|static|property|class|method|function|mysql|redis|mongo"`
    echo $content | mail -s 'PHP 日志监控' guanhua2011@gmail.com
fi

mysql与mongo比对

学习mongo时，将mongo语句与mysql做下比对

CREATE TABLE USERS (a Number, b Number)                 
隐式的创建，或 MongoDB::createCollection() . 

INSERT INTO USERS VALUES(1,1)                                   
$db->users->insert(array("a" => 1, "b" => 1)); 

SELECT a,b FROM users                                                    
$db->users->find(array(), array("a" => 1, "b" => 1)); 

SELECT * FROM users WHERE age=33                            
$db->users->find(array("age" => 33)); 

SELECT a,b FROM users WHERE age=33                         
$db->users->find(array("age" => 33), array("a" => 1, "b" => 1)); 

SELECT a,b FROM users WHERE age=33 ORDER BY name       
$db->users->find(array("age" => 33), array("a" => 1, "b" => 1))->sort(array("name" => 1)); 

SELECT * FROM users WHERE age>33                             
$db->users->find(array("age" => array('$gt' => 33))); 

SELECT * FROM users WHERE age<33                             
$db->users->find(array("age" => array('$lt' => 33))); 

SELECT * FROM users WHERE name LIKE "%Joe%"        
$db->users->find(array("name" => new MongoRegex("/Joe/"))); 

SELECT * FROM users WHERE name LIKE "Joe%"            
$db->users->find(array("name" => new MongoRegex("/^Joe/"))); 

SELECT * FROM users WHERE age>33 AND age<=40       
$db->users->find(array("age" => array('$gt' => 33, '$lte' => 40))); 

SELECT * FROM users ORDER BY name DESC                  
$db->users->find()->sort(array("name" => -1)); 

CREATE INDEX myindexname ON users(name)                   
$db->users->ensureIndex(array("name" => 1)); 

CREATE INDEX myindexname ON users(name,ts DESC)     
$db->users->ensureIndex(array("name" => 1, "ts" => -1)); 

SELECT * FROM users WHERE a=1 and b='q'                      
$db->users->find(array("a" => 1, "b" => "q")); 

SELECT * FROM users LIMIT 20, 10                                      
$db->users->find()->limit(10)->skip(20); 

SELECT * FROM users WHERE a=1 or b=2                          
$db->users->find(array('$or' => array(array("a" => 1), array("b" => 2)))); 

SELECT * FROM users LIMIT 1                                              
$db->users->find()->limit(1); 

EXPLAIN SELECT * FROM users WHERE z=3                      
$db->users->find(array("z" => 3))->explain() 

SELECT DISTINCT last_name FROM users                           
$db->command(array("distinct" => "users", "key" => "last_name")); 

SELECT COUNT(*) FROM users                                           
$db->users->count(); 

SELECT COUNT(*) FROM users where AGE > 30               
$db->users->find(array("age" => array('$gt' => 30)))->count(); 

SELECT COUNT(AGE) from users                                
$db->users->find(array("age" => array('$exists' => true)))->count(); 

UPDATE users SET a=1 WHERE b='q'                           
$db->users->update(array("b" => "q"), array('$set' => array("a" => 1))); 

UPDATE users SET a=a+2 WHERE b='q'                      
$db->users->update(array("b" => "q"), array('$inc' => array("a" => 2))); 

DELETE FROM users WHERE z="abc"                           
$db->users->remove(array("z" => "abc"));

接口操作时间统计

关于服务器接口访问的操作时间简单分析，通过awk实现。

cat server.log.20141201 | awk '{gsub(/[^0-9]/,"",$8);a[$3]+=$8;b[$3]+=1} END {for( i in a) {printf("%d\t%5.2f\t%s\n",b[i], a[i]/b[i],i) }}'

获取某个操作包好某项信息的统计信息

cat server.log.20141201 | awk '{i=index($3,"actionID");if(i>0){s=substr($3,i+11,1);a[s]+=1;}}END{for(ii in a) printf("%s\t%d\n",ii, a[ii]);}'
